# 版本发布说明

Aries AI 的版本发布与更新通知系统，实现 GitHub Releases 自动拉取、静默版本检查、系统通知、APK 下载与镜像加速等完整更新链路。

## 概述

版本发布说明系统是 Aries AI 内建的更新管理模块，位于 `feature/updates/` 模块中。它负责从 GitHub Releases 拉取版本信息，在应用启动时进行静默版本检查，在"关于"页面提供手动更新检查入口，以及在"更新历史"页面展示完整的版本列表。

**核心设计意图：**

- **双重数据源**：优先使用 GitHub REST API 获取结构化版本数据，当 API 限流或不可用时，自动降级到 Atom Feed 作为备用数据源，确保更新检查的高可用性。
- **静默 + 主动双通道**：应用启动时自动执行静默检查（每 6 小时一次），用户也可在"关于"页面手动触发检查，兼顾时效性与用户体验。
- **镜像加速下载**：针对国内网络环境，自动探测镜像站可用性，按延迟排序提供最优下载链接，降低 APK 下载失败率。
- **防重复通知**：基于 `SharedPreferences` 记录已通知版本号，避免每次启动都弹出相同的更新通知。

## 关键组件详解

### ReleaseRepository — 版本数据仓库

`ReleaseRepository` 是版本数据的核心获取层。它封装了对 GitHub Releases 的全部访问逻辑，并提供了优雅的降级策略。

**核心设计：**

- **`fetchReleasePage()`**：分页获取 GitHub Release 列表，自动过滤草稿（draft），将 GitHub API 原始模型 `GitHubRelease` 转换为内部使用的 `ReleaseEntry`。如果 APK 资产匹配 `UpdateConfig.APK_ASSET_NAME` 则优先使用，否则回退到第一个 `.apk` 文件。

```kotlin

    return runCatching {
        val releases = GitHubApiClient.listReleases(owner = owner, repo = repo, page = page, perPage = perPage)
        releases
            .asSequence()
            .filter { !it.draft }
            .map { r ->
                // 版本标签处理、APK 资产匹配
                ReleaseEntry(/* ... */)
            }
            .toList()
    }
}
```
> Source: [ReleaseRepository.kt](https://github.com/ZG0704666/Aries-AI/blob/main/feature/updates/src/main/java/com/ai/phoneagent/updates/ReleaseRepository.kt#L7-L37)

- **`fetchLatestReleaseResilient()`**：高可用兜底方法。先尝试 GitHub API 最多 3 次（指数退避），全部失败后降级到 `AtomReleaseFallback` 解析 GitHub 的 Atom Feed。这确保了即使 GitHub API 限流，用户仍然可以获得版本信息。

```kotlin
suspend fun fetchLatestReleaseResilient(includePrerelease: Boolean): Result<ReleaseEntry?> {
    var lastErr: Throwable? = null
    var delayMs = 500L
    repeat(3) { attempt ->
        val result = fetchLatestRelease(includePrerelease)
        if (result.isSuccess) return result
        lastErr = result.exceptionOrNull()
        if (attempt < 2) {
            kotlinx.coroutines.delay(delayMs)
            delayMs *= 2
        }
    }
    // 降级到 Atom Feed
    val atom = AtomReleaseFallback.fetchLatest(owner = owner, repo = repo)
    // ...
}
```

### GitHubApiClient — GitHub REST API 客户端

基于 OkHttp 的 GitHub REST API 客户端，使用 `kotlinx.serialization` 进行 JSON 反序列化。

- 支持 GitHub Personal Access Token（`github_pat_*` 前缀用 Bearer 认证，其他用 token 认证）
- Debug 模式下开启 HTTP 日志拦截器
- 超时配置：连接 20s、读取 30s、写入 30s、整体 60s

```kotlin
private val okHttpClient: OkHttpClient by lazy {
    val headerInterceptor = Interceptor { chain ->
        val reqBuilder = chain.request().newBuilder()
            .header("User-Agent", "PhoneAgent")
            .header("Accept", "application/vnd.github+json")
        buildAuthHeader(BuildConfig.GITHUB_TOKEN)?.let { auth ->
            reqBuilder.header("Authorization", auth)
        }
        chain.proceed(reqBuilder.build())
    }
    OkHttpClient.Builder()
        .addInterceptor(headerInterceptor)
        .retryOnConnectionFailure(true)
        .connectTimeout(20, TimeUnit.SECONDS)
        .readTimeout(30, TimeUnit.SECONDS)
        .build()
}
```

### AtomReleaseFallback — Atom Feed 备用数据源

当 GitHub REST API 不可用（如触发限流）时，`AtomReleaseFallback` 通过解析 GitHub Releases 页面的 Atom Feed XML 来获取版本信息。

- 从 `<entry>` 元素中提取 `title`、`updated`、`link href`、`content`/`summary`
- 自动构造 APK 下载链接

```kotlin
}
```

### UpdateStartupCoordinator — 启动静默检查协调器

应用启动时的入口协调器，执行"缓存优先 + 远程拉取"的两阶段检查策略：

1. **缓存优先**：先从 `UpdateStore` 加载上次缓存的版本信息，如果比当前版本新则立即通知
2. **远程拉取**：检查距上次远程拉取是否超过 6 小时，超过则通过 `ReleaseRepository.fetchLatestReleaseResilient()` 拉取最新版本

```kotlin
fun silentCheckOnLaunch(activity: AppCompatActivity) {
    val currentVersion = /* 获取当前应用版本 */
    // 第一步：缓存通知
    notifyFromCacheIfNeeded(activity, currentVersion)
    // 第二步：检查静默间隔
    if (!UpdateStore.shouldSilentCheck(activity, nowMs = now, intervalMs = SILENT_CHECK_INTERVAL_MS)) return
    UpdateStore.markSilentChecked(activity, nowMs = now)
    // 第三步：远程拉取
    activity.lifecycleScope.launch {
        val result = withContext(Dispatchers.IO) {
            ReleaseRepository().fetchLatestReleaseResilient(includePrerelease = false)
        }
        result.onSuccess { latest ->
            if (VersionComparator.compare(latest.version, currentVersion) <= 0) return@onSuccess
            UpdateStore.saveLatest(activity, latest)
            notifyIfNeeded(activity, latest)
        }
    }
}
```

### VersionComparator — 语义版本号比较

从 `core/common` 模块提供的通用版本号比较工具，解析 `major.minor.patch[+build]` 格式的版本号，按 `major → minor → patch → build` 优先级逐级比较。

```kotlin
object VersionComparator {
    fun compare(v1: String, v2: String): Int {
        val p1 = parse(v1)
        val p2 = parse(v2)
        if (p1.major != p2.major) return p1.major.compareTo(p2.major)
        if (p1.minor != p2.minor) return p1.minor.compareTo(p2.minor)
        if (p1.patch != p2.patch) return p1.patch.compareTo(p2.patch)
        return p1.build.compareTo(p2.build)
    }
}
```

```kotlin
private fun probeOneUrl(url: String, timeoutMs: Int): ProbeResult {
    val startNs = System.nanoTime()
    return try {
        val code = requestOnce(url, timeoutMs, method = "HEAD")
            ?: requestOnce(url, timeoutMs, method = "GET")
            ?: -1
        val costMs = (System.nanoTime() - startNs) / 1_000_000
        val ok = code in 200..399
        ProbeResult(ok = ok, latencyMs = costMs, error = if (ok) null else "HTTP $code")
    } catch (e: Exception) {
        ProbeResult(ok = false, latencyMs = null, error = e.javaClass.simpleName)
    }
}
```

```kotlin
fun checkForUpdates() {

        val result = withContext(Dispatchers.IO) {

        if (latest != null && VersionComparator.compare(latest.version, currentVersion) > 0) {
            UpdateStore.saveLatest(context, latest)
            _uiState.update { it.copy(updateDialogState = UpdateDialogState(latest)) }
        } else {
            _uiState.update { it.copy(upToDateDialogState = UpToDateDialogState(currentVersion)) }
        }
    }
}
```
