# Aries AI - 项目概述与核心特性

## 一、项目简介

**Aries AI** 是一个开源的 Android UI 自动化引擎，通过接入大语言模型（LLM），让 AI 理解屏幕内容并自动执行复杂任务。项目代号 "Aries AI"，包名为 `com.ai.phoneagent`，当前版本为 **v1.4.2-xyla.alpha (versionCode: 17)**。

### 核心亮点

| 特性 | 说明 |
|------|------|
| 完全隔离 | 独创虚拟屏技术，焦点 100% 隔离，主屏幕零干扰 |
| 性能领先 | 响应时间 1.8s，成功率 94%，比同类产品快 44% |
| 完全开源 | 代码透明（AGPL-3.0 协议），可自由定制和扩展 |
| 模型灵活 | 兼容 OpenAI 接口，支持 20+ 种大模型 |

### 典型应用场景

- 自动化订票（火车票、机票）
- 餐厅预订
- 批量操作
- 应用测试
- 数据采集

## 二、核心特性详解

### 1. 虚拟屏完全隔离技术

Aries AI 采用业界领先的虚拟屏隔离方案，实现真正的后台自动化：

- **焦点完全隔离**：100ms 高频强制执行，物理按键永不误触虚拟屏
- **OpenGL 分发**：同时支持截图和预览，无需切换 Surface
- **智能截图**：自动等待非黑帧，避免截取空白画面
- **IME 隔离**：防止键盘焦点死锁，更稳定可靠

相比同类产品，Aries AI 的焦点恢复速度快 20 倍（100ms vs 2s），实现真正的完全隔离。

### 2. 性能优化数据

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 响应时间 | 3.2s | 1.8s | 44% ↑ |
| 截图大小 | 250KB | 85KB | 66% ↓ |
| 成功率 | 82% | 94% | 12% ↑ |

优化技术：
- 智能截图压缩（85% 质量，16 像素对齐）
- 流式早停机制（识别到动作立即执行）
- 并行状态采集（截图和 UI 树同时获取）
- 异步输入注入（通过 displayId 定向注入）

### 3. 多模型支持

兼容所有 OpenAI 接口标准的视觉语言模型：

- **国内模型**：智谱 GLM、DeepSeek、Qwen、MiniMax、百川、零一万物等
- **国际模型**：OpenAI GPT-4V、Claude、Gemini 等
- **开源模型**：LLaVA、CogVLM、Qwen-VL 等
- **API 聚合平台**：硅基流动、API2D、OpenRouter 等
- **自部署模型**：任何兼容 OpenAI 接口的自部署服务
- **本地模型**：通过 MNN 推理引擎支持本地运行 Qwen 等模型

### 4. 三种使用方式

1. **主界面对话**：语音或文本输入任务指令
2. **预设任务**：餐厅预订、火车票预订、机票预订、自定义任务
3. **虚拟屏幕模式**（推荐）：后台运行，实时预览

## 三、技术栈

| 类别 | 技术 |
|------|------|
| 编程语言 | Kotlin 2.2.21, Java, C++ |
| UI 框架 | Jetpack Compose + Material 3 |
| 依赖注入 | Koin (3.5.6) |
| 本地存储 | Room 数据库 + DataStore Preferences |
| 网络 | OkHttp 4.12 + Gson |
| 图片加载 | Coil 2 + Coil 3 |
| 序列化 | Kotlinx Serialization |
| Markdown 渲染 | JetBrains Markdown Parser + QuickJS |
| 语音识别 | Sherpa-NCNN (本地离线) |
| 本地推理 | MNN (阿里巴巴开源) |
| 虚拟屏 | Shizuku API + HiddenApiBypass |
| 后台任务 | WorkManager |
| 遥测 | Cloudflare Workers + D1 数据库 |
| 构建工具 | Gradle 8.13 + AGP 8.13.2 |
| 最低版本 | Android 11 (API 30) |
| 目标版本 | Android 16 (API 36) |
