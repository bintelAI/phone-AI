# Aries AI - 业务功能分析

## 一、核心业务流程

### 1. 自动化任务执行流程

```
用户输入任务指令 → 智能应用启动 → Agent 主循环 → 任务完成
```

**详细流程说明**：

1. **用户输入**：用户通过文本或语音输入任务指令（如"帮我订一张明天北京到上海的机票"）
2. **智能应用启动**：系统自动识别需要打开的应用，智能启动目标应用（支持无障碍和 Shizuku 两种通道）
3. **Agent 主循环**：
   - 并行获取截图和 UI 树（状态采集）
   - 构建消息发送给 LLM 模型（含截图+UI树+历史上下文）
   - 解析模型输出（思考过程 + 执行动作）
   - 执行动作（点击/滑动/输入/返回等）
   - 等待界面稳定
   - 循环直到任务完成或达到最大步数
4. **任务完成**：返回执行结果（成功/失败、步骤数、消息）

### 2. 核心业务模块

#### 2.1 对话交互系统
- 文本输入/语音输入
- 流式 Markdown 渲染
- 对话历史管理
- 会话存储（Room 数据库）
- 附件管理（图片、文件）

#### 2.2 AI 模型接入层
- OpenAI 兼容接口适配
- 多模型切换（云端/本地）
- 流式输出 + 早停机制
- 自动重试与故障恢复
- 本地 MNN 推理引擎

#### 2.3 自动化引擎
- Agent 主循环控制
- UI 状态采集（截图 + 无障碍树）
- 模型输出解析（动作解析器）
- 动作执行器（点击/滑动/输入/返回/启动应用等）
- 动作修复与重试机制
- 进度跟踪与通知

#### 2.4 虚拟屏隔离系统
- VirtualDisplay 创建/销毁
- OpenGL 帧分发（截图 + 预览）
- 焦点强制执行（100ms 频率）
- IME 死锁防护
- 输入注入（通过 displayId 定向）

#### 2.5 AI 工具框架
- 工具注册与权限管理
- 文件工具执行器
- 网络工具执行器
- 应用包名映射管理

#### 2.6 权限管理体系
- 无障碍服务权限引导
- Shizuku 权限授权
- 敏感操作二次确认
- 工具权限分级管理

#### 2.7 用户系统与遥测
- Logto OIDC 认证
- 会员/订阅管理
- 遥测心跳上报（Cloudflare Workers）
- 使用统计仪表盘

| 功能 | 说明 |
|------|------|
| 智能应用启动 | 自动识别并打开目标应用 |

| 进度通知 | 自动化任务状态实时推送 |
| 危险操作防护 | 敏感关键词检测和二次确认 |

| 功能 | 说明 |

## 四、非功能性需求

### 性能要求
- 单步操作响应时间 < 2s
- 截图压缩后 < 150KB
- 虚拟屏焦点恢复 < 100ms
- 任务成功率 > 90%

### 兼容性
- 最低 Android 11 (API 30)
- 目标 Android 16 (API 36)
- 支持 arm64-v8a 架构
- 适配不同厂商 ROM

### 安全性
- API Key 本地安全存储
- 敏感操作二次确认
- 支付密码/验证码等敏感内容检测
- 无网络请求时本地模式运行

### 可维护性
- Koin 依赖注入（模块化）
- MVVM 架构（ViewModel + Repository）
- 数据层通过 Room + DataStore 持久化
- DI 模块分离（AppModule、DataModule、NetworkModule、UiModule）
