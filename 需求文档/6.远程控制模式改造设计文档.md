# Aries AI — 远程控制模式改造设计文档（简化版）

> **设计目标**：在完全不破坏现有代码架构的前提下，新增「远程控制模式」，实现登录 → WebSocket 长连接 → 服务端任务下发 → 手机自动执行的闭环。
>
> **认证方式**：仅 **Casdoor OIDC**（无需 Logto 和策略模式）。手机 App 启动即进入登录页，支持「离线使用」兜底。

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **零侵入** | 不修改任何现有 `.kt` 文件（除 Koin 模块注册外） |
| **独立包** | 所有新代码位于 `com.ai.phoneagent.remote` 包下 |
| **可选离线** | 登录页底部提供「离线使用」入口，不登录也可使用本地功能 |
| **渐进式** | 第一阶段实现核心链路，后续迭代增强 |

### 5.3 CasdoorAuthManager — Casdoor OIDC 登录管理器

```kotlin
package com.ai.phoneagent.remote

/**
 * Casdoor OIDC 认证管理器。
 *
 * 唯一的认证方式。使用 Custom Tab + OkHttp 实现 OIDC 授权码流程，
 * 无需引入任何额外依赖。
 *
 * 流程：
 *   1. 打开 Casdoor 授权页（Custom Tab）
 *   2. 用户登录 → 回调返回 code
 *   3. OkHttp POST 换 token
 *   4. 获取 userInfo
 *   5. 持久化 token
 *
 * 状态通过 DataStore 持久化，App 重启后自动恢复。
 */
class CasdoorAuthManager(
    private val okHttpClient: OkHttpClient,
    private val appPreferencesRepository: AppPreferencesRepository,
    private val appContext: Context,
) {
    /** 登录结果 */
    data class LoginResult(
        val success: Boolean,
        val accessToken: String = "",
        val refreshToken: String = "",
        val userInfo: CasdoorUserInfo? = null,
        val message: String = "",
    )

    data class CasdoorUserInfo(
        val id: String,
        val name: String,
        val email: String?,
        val avatar: String?,
    )

    /** 执行 Casdoor OIDC 登录 */
    suspend fun signIn(activity: Activity): LoginResult

    /** 静默刷新 token */
    suspend fun refreshToken(): Boolean

    /** 获取有效 accessToken（自动尝试刷新） */
    suspend fun getAccessToken(): String?

    /** 获取设备唯一 ID */
    fun getDeviceId(): String

    /** 是否已登录 */
    fun isLoggedIn(): Boolean

    /** 退出登录 */
    suspend fun signOut()
}
```

**Casdoor OIDC 授权码流程实现细节**：

```
Step 1: 构建授权 URL
──────────────────────
GET {CASDOOR_ENDPOINT}/login/oauth/authorize
  ?client_id={CLIENT_ID}
  &redirect_uri={CASDOOR_REDIRECT_URI}
  &response_type=code
  &scope=openid+profile+email
  &state={SHA256随机字符串}

Step 2: 通过 Android Custom Tabs 打开
───────────────────────────────────────
val builder = CustomTabsIntent.Builder()
builder.build().launchUrl(activity, authorizeUri)

Step 3: CasdoorCallbackActivity 接收回调
────────────────────────────────────────
com.ai.phoneagent://oauth2redirect
  ?code=xxx&state=xxx

→ CasdoorCallbackActivity 通过 LocalBroadcast 或 SharedFlow
  将 code 发送回 CasdoorAuthManager

Step 4: code 换 token（OkHttp POST）
────────────────────────────────────
POST {CASDOOR_ENDPOINT}/api/login/oauth/access_token
Content-Type: application/json
{
  "client_id": "{CLIENT_ID}",
  "client_secret": "{CLIENT_SECRET}",
  "code": "{CODE}",
  "grant_type": "authorization_code",
  "redirect_uri": "{CASDOOR_REDIRECT_URI}"
}
Response: { "access_token": "...", "refresh_token": "...", ... }

Step 5: 获取 userInfo
─────────────────────
GET {CASDOOR_ENDPOINT}/api/userinfo
Authorization: Bearer {access_token}
```

### 状态流转逻辑

```kotlin
@Composable
fun LoginGateScreen(
    navController: NavController,
    casdoorAuthManager: CasdoorAuthManager,
) {
    // 读取持久化的登录状态
    val isLoggedIn by remember { mutableStateOf(casdoorAuthManager.isLoggedIn()) }
    val hasChosenOffline by remember { /* DataStore 读取 */ }

    LaunchedEffect(Unit) {
        if (isLoggedIn || hasChosenOffline) {
            navController.navigate(Routes.Home.route) {
                popUpTo(Routes.Login.route) { inclusive = true }
            }
        }
    }

    // 登录按钮 → 打开 Casdoor OIDC 授权页
    // 离线按钮 → 持久化离线选择 → 跳转 Home
    // 两种情况都不再回到 LoginGateScreen（popUpTo inclusive）
}
```

## 八、离线模式行为

| 功能 | 登录状态 | 离线状态 |
|------|----------|----------|
| 主对话界面 | ✅ 可用 | ✅ 可用 |
| 本地自动化 | ✅ 可用 | ✅ 可用 |
| 设置页面 | ✅ 可用 | ✅ 可用 |
| 远程控制 | ✅ 可用（WebSocket 连接） | ❌ 隐藏入口 |
| WebSocket 连接 | ✅ 自动连接 | ❌ 不启动 |
| 任务下发 | ✅ 接收执行 | ❌ 不可用 |
| OTA 更新 | ✅ 可用 | ✅ 可用 |

**离线模式实现**：
- 在 `AppPreferencesRepository` 中新增 `hasChosenOffline` 标志
- 首页/设置中提供「登录/绑定账号」入口，用户可以随时从离线切换到登录模式
- 远程控制入口在离线模式下隐藏

## 十三、安全设计

| 安全措施 | 实现方式 |
|----------|----------|
| 传输加密 | WebSocket over WSS（TLS） |
| 设备认证 | 连接时发送 token + deviceId 校验 |
| 令牌刷新 | Casdoor refresh_token 自动刷新 |
| Authorization Code Flow | 标准 OIDC 授权码流程，token 不暴露在浏览器端 |
| PKCE | 建议启用 PKCE 扩展，防止授权码拦截攻击 |
| Token 加密存储 | accessToken / refreshToken 通过 DataStore 加密存储 |
| 最小权限 | 不申请额外权限 |

## 十四、错误处理

| 场景 | 处理策略 |
|------|----------|
| Casdoor 授权失败 | 显示错误原因，允许重试 |
| token 过期 | 自动 refresh_token 刷新，失败则跳回登录页 |
| WebSocket 断连 | 自动重连（指数退避，最大 10 次） |
| 任务中段连 | 标记暂停，重连后发送状态，等待服务端决定 |
| 多端同时控制 | 服务端保证单设备单活跃连接 |
| 设备离线 | 服务端缓存任务，上线后补发 |
